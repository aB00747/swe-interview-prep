# Design Patterns

## Creational Patterns
- **Singleton**: One instance of a class
- **Factory**: Create objects without specifying exact class
- **Abstract Factory**: Factory of factories
- **Builder**: Step-by-step object construction
- **Prototype**: Clone existing objects

## Structural Patterns
- **Adapter**: Convert interface of one class to another
- **Decorator**: Add behavior dynamically
- **Facade**: Simplified interface to complex subsystem
- **Proxy**: Placeholder for another object

## Behavioral Patterns
- **Observer**: Notify dependents of state changes
- **Strategy**: Swap algorithms at runtime
- **Command**: Encapsulate request as object
- **State**: Change behavior based on internal state
- **Iterator**: Sequential access to collection elements
