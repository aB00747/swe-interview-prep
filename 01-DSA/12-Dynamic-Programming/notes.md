# Dynamic Programming

## Core Idea
- Break problem into overlapping subproblems
- Store results to avoid recomputation (memoization / tabulation)

## Approaches
- **Top-Down** (Recursion + Memoization)
- **Bottom-Up** (Tabulation)

## Common DP Patterns
| Pattern | Examples |
|---------|----------|
| Take / Not Take | 0/1 Knapsack, House Robber, Subset Sum |
| Infinite Supply | Coin Change, Unbounded Knapsack |
| DP on Strings | LCS, Edit Distance, Palindrome |
| DP on Grids | Unique Paths, Min Path Sum |
| DP on Stocks | Buy/Sell Stock I-IV |
| Partition DP | Matrix Chain, Burst Balloons |
| DP on Trees | House Robber III, Diameter |

## Steps to Solve
1. Define state (what changes between subproblems?)
2. Write recurrence relation
3. Identify base cases
4. Decide top-down vs bottom-up
5. Optimize space if possible
